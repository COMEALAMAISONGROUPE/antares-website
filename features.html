🔒 Safe Gate
Score-independent hard block — cannot be bypassed
The Safe Gate is a hardcoded set of non-negotiable checks that run alongside the scoring system. A token can score 950/1000 and still be fully blocked if a Safe Gate condition is met. This prevents edge cases where a manipulated token looks statistically safe but contains a critical trap. It is the last line of defense — and the most important one. As of v6.0.0, the SAFE verdict additionally requires a score of ≥ 850 pts with zero outstanding critical flags — conditional override logic closes all previously known bypass edge cases.

Real-Time Scanning

PASSIVEAUTOMATIC<2s

Antares runs in the background. You never paste a contract address manually. Every time a Solana address appears on a page you visit — Twitter, Telegram web, Discord, any DEX — the scan fires automatically and the overlay appears in under 2 seconds. The Chrome extension icon simultaneously updates with a ✓, !, or ✗ badge — the verdict is visible at a glance without opening the popup. If a previously scanned token’s risk level escalates, Antares fires a Chrome notification automatically.

How it works
  • Content script watches DOM mutations and URL changes
  • Solana address regex matches base58 strings of 32–44 chars
  • Debounced to avoid duplicate scans on the same address
  • All 7 API calls run in parallel via Promise.allSettled
  • Overlay injects into page DOM with isolated CSS

7 Data Sources

DEXSCREENERRUGCHECKGOPLUSHELIUSSOLSCANGECKOTERMINALIDENTITY REGISTRY

No single source is trusted alone. Antares queries 7 independent APIs simultaneously and cross-validates results. Each source covers a different attack surface. If sources conflict, the most conservative verdict wins. Full details on the Sources page. A burst rate limiter (5 req / 10s) queues scans gracefully during high-frequency browsing without dropping any result.

Source roles
  • DexScreener — price, liquidity, OHLCV chart data
  • RugCheck — LP status, metadata, bundler, risk flags
  • GoPlus — honeypot, proxy, blacklist, sell restrictions
  • Helius RPC — on-chain mint/freeze authority, supply
  • Solscan — holder count, concentration, token age
  • GeckoTerminal — OHLCV candles 5m / 1h / 6h / 24h (Chart Intelligence)
  • Internal Registry — identity validation, copycat & unicode spoofing detection

Chart Intelligence

OHLCVPUMP DETECTIONWASH TRADING

Most rug detection tools ignore price action. Antares analyses OHLCV candle data from GeckoTerminal across four time windows (5m, 1h, 6h, 24h) to detect manipulation patterns that precede a rug: parabolic pumps, blow-off tops, and wash trading designed to fake volume.

Detected patterns
  • Parabolic pump — price >500% in <1h with no retracement
  • Blow-off top — spike followed by immediate 80%+ crash
  • Wash trading — volume/liquidity ratio anomaly (>10x)
  • Abnormal B/S ratio — 100% buys in 5m window (coordinated)
  • Dead chart — no recent candles despite listed token

Identity Check

COPYCATIMPERSONATIONBRANDING

A large number of scams impersonate established tokens with nearly identical names and symbols. Antares cross-references every token name and ticker against known projects. Copycat detection covers exact matches, unicode spoofing (e.g. вs vs v), and zero-width character tricks.

Detection methods
  • Exact name match against known token registry
  • Levenshtein distance check (1–2 char variations)
  • Unicode normalization to catch visual spoofing
  • Symbol collision detection (e.g. two tokens named WIF)
  • Social link absence penalty when branding is copied
