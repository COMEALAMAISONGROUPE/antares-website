Data Sources

7 APIS. ONE VERDICT.

Antares never trusts a single source. Seven independent APIs are queried in parallel on every scan. Each covers a distinct attack surface. Results are cross-validated — when sources conflict, the most conservative verdict wins. A failed source is skipped gracefully without blocking the scan.

01
DexScreener
Price · Liquidity · Chart
PRICELIQUIDITYOHLCVTXNS
Why it matters

DexScreener is the primary market data layer. It provides the token's real-time price, market cap, liquidity depth across all active pairs, and granular transaction counts across 5m, 1h, 6h, and 24h windows. Real-time price action and transaction counts across 5m, 1h, 6h, and 24h windows feed into liquidity and volume scoring. OHLCV candle data is now sourced from GeckoTerminal for Chart Intelligence. A token with no DexScreener pair is likely not yet indexed or is too illiquid to trade safely.

02
RugCheck
LP · Metadata · Risks
LP STATUSFREEZEBUNDLERMETADATA
Why it matters

RugCheck specialises in Solana token audit. It checks whether LP tokens have been burned or locked (preventing a rug via liquidity removal), whether the token metadata is mutable (can the name/image be changed post-launch?), and whether mint or freeze authorities are still active. It also runs bundler detection — identifying tokens launched via coordinated bots that pre-buy supply before public trading. Its risk score and flag list feed directly into the Safe Gate and scoring engine.

03
GoPlus
Honeypot · Contract · Blacklist
HONEYPOTPROXYBLACKLISTSELL LOCK
Why it matters

GoPlus is the contract security layer. It performs a simulated transaction to determine whether you can actually sell the token after buying — the classic honeypot trap. It also detects proxy contracts that can be upgraded to steal funds, blacklist functions that block specific wallets from selling, transfer restrictions, and hidden fees that make trading economically impossible. A honeypot detection from GoPlus is an immediate Safe Gate trigger regardless of other scores.

04
Helius RPC
On-Chain · Authority · Supply
MINT AUTHFREEZE AUTHSUPPLYACCOUNTS
Why it matters

Helius provides a premium Solana RPC endpoint used to make direct on-chain calls. This is the ground truth layer: it bypasses any API aggregator and reads state directly from the Solana blockchain. Antares uses it to verify mint authority (can the developer infinitely print more tokens?), freeze authority (can wallets be locked, preventing selling?), current token supply, and active token accounts. These checks cannot be faked or manipulated by a bad actor controlling metadata.

05
Solscan
Holders · Age · Volume
HOLDERSTOKEN AGEVOLUMEDISTRIBUTION
Why it matters

Solscan is the holder distribution and token history layer. It provides the number of unique holders, top wallet percentages for concentration analysis, token creation timestamp (age is a trust signal), and 24h trading volume independent from DexScreener for cross-validation. High concentration in the top 10 wallets is a strong dump signal. A token created less than an hour ago with 12 holders and 90% in one wallet is a textbook pre-rug setup — Solscan catches this pattern.

GeckoTerminal

Why it matters

GeckoTerminal is the OHLCV candle data provider for the Chart Intelligence layer (L6). Since Phase 1, it replaces DexScreener as the source for raw price candles across 5m, 1h, 6h, and 24h windows. Pump-and-dump detection, blow-off top analysis, and wash trading identification all depend on this feed. Without valid candle data, L6 scoring is skipped gracefully rather than blocking the scan.

07
Identity Registry
Token Identity · Copycat Detection
IDENTITYSPOOFINGCOPYCAT
Why it matters

The internal identity registry powers the Identity Validation layer (L7). It cross-references every token name and ticker symbol against a curated database of known legitimate projects. It catches exact name clones, unicode spoofing attacks (Cyrillic characters that look identical to Latin ones), zero-width character tricks, and ticker symbol collisions — all common tactics used by copycat tokens to steal a project’s reputation.

Cross-Validation Logic
What happens when sources conflict
Conservative wins
When two sources give conflicting signals on the same check (e.g. RugCheck says mint auth is revoked but Helius RPC says it is still active), Antares always takes the more dangerous result. The on-chain RPC call is treated as ground truth over aggregated APIs.
Graceful degradation
If a source times out or returns an error, it is skipped via Promise.allSettled. The scan continues with the remaining sources. The confidence score drops proportionally: missing 2 of 7 sources reduces confidence to 71%. The verdict still fires, but the user sees reduced confidence.
Safe Gate cannot be skipped
Safe Gate checks require only one confirming source. If GoPlus alone detects a honeypot, the gate triggers — even if the other 4 sources are unavailable. Critical safety checks are never gated behind multi-source consensus.
