Scoring Engine

HOW THE SCORE IS BUILT

Six independent analysis layers each contribute weighted deductions to a 0–1000 base score. A hardcoded Safe Gate overrides everything — critical flags block the token regardless of numeric score. This page covers the technical scoring logic. For what you see in the browser, visit Features.

Analysis Layers
6 independent checks — each weighted in the final score
L1
Contract Security
GoPlus · RugCheck
Checks for honeypot logic, sell restrictions, proxy contracts, blacklist functions, and freeze authorities embedded in the token contract. A single critical flag here activates the Safe Gate.
Up to −400 pts on critical fail · Safe Gate trigger
L2
On-Chain Data
RugCheck · Helius RPC
Verifies mint authority (can supply be inflated?), freeze authority (can wallets be locked?), metadata mutability, and token creation context directly from the Solana blockchain.
−200 pts mint auth · −150 pts freeze auth · −80 pts mutable metadata
L3
Liquidity & Pool
DexScreener
Evaluates total liquidity depth, liquidity-to-market-cap ratio, LP burn/lock status, and number of active trading pairs. Low liquidity makes exit manipulation trivially easy.
Graduated −50 to −200 pts based on liquidity depth
L4
Holder Distribution
RugCheck · Solscan
Analyzes top-10 wallet concentration, developer wallet size, and holder count. High concentration in few wallets signals coordinated dump risk.
−100 pts if top-10 >60% · −60 pts if dev wallet >10%
L5
Identity Risk
DexScreener Metadata
Cross-references token name, symbol, and branding against known tokens and copycat patterns. Detects impersonation of established projects like WIF, BONK, PEPE, etc.
−120 pts copycat detected · −40 pts no social presence
L6
Chart Intelligence
DexScreener OHLCV
Detects parabolic pump-and-dump patterns, blow-off tops (sudden spike + crash), wash trading (artificial volume), and abnormal buy/sell ratios across 5m, 1h, 6h, and 24h windows.
−150 pts blow-off top · −80 pts wash trading detected
🔒 Safe Gate
Score-independent hard block
The Safe Gate is a set of non-negotiable checks that bypass the scoring system entirely. If any condition below is detected, the token is blocked regardless of its numeric score — even a 900/1000 token gets blocked if a honeypot is found. Requires only one confirming source to trigger.

Scoring System
How 0–1000 points are distributed
Base Score
Every token starts at 1000 points. Each risk flag deducts a weighted amount. Positive signals (LP burned, no mint authority, strong liquidity) can partially restore points. The final score reflects net confidence after all deductions.
Verdict Thresholds
SAFE — 700–1000 pts

CAUTION — 400–699 pts

DANGER — 100–399 pts

RUG PULL — 0–99 pts or Safe Gate triggered
Confidence Score
A separate 0–100% confidence metric reflects data completeness. If only 2 of 5 sources return data, confidence is lower. High confidence means the verdict is based on full cross-source validation.
Scan Speed
All 5 sources are queried in parallel via Promise.allSettled. Failed sources are gracefully skipped without blocking the scan. Average scan completes in under 2 seconds for indexed tokens.
